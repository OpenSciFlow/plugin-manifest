# Local agent contract

This document defines how an OpenSciFlow-compatible local agent should use `opensciflow.yaml`.

## Core rule

The agent may execute only reviewed command templates from manifests. It must not execute arbitrary shell code generated by an LLM.

## Agent responsibilities

Before execution, the agent must:

- parse the manifest;
- validate schema version;
- resolve required inputs;
- show license, citation, model weights, remote services, hardware, and limitations;
- check environment readiness;
- render command templates using only declared inputs and output paths;
- ask for user approval when execution is not a dry run.

During execution, the agent must:

- create a stable run directory;
- stream or capture logs;
- avoid writing outside the run directory except for declared cache/download paths;
- fail closed on missing inputs, missing weights, unsupported hardware, or unresolved command variables;
- record timestamps and return codes.

After execution, the agent must:

- register artifacts;
- compute output hashes where feasible;
- write a run record;
- render a report only from recorded artifacts;
- include citations and limitations in the report.

## Refusal conditions

The agent must refuse to execute if:

- the manifest fails schema validation;
- required inputs are missing;
- command template references undeclared variables;
- license or model-weight terms are unknown for a required model;
- a remote service is required but not disclosed;
- the workflow asks for clinical, diagnostic, or drug-efficacy claims;
- the command was generated by an LLM instead of coming from a reviewed manifest.

## Command rendering rules

Allowed placeholders:

```text
{inputs.<name>}
{outputs_dir}
{run_dir}
{cache_dir}
```

v0.2 should reject:

- shell operators outside reviewed templates;
- undeclared environment variables;
- relative paths escaping the run directory;
- commands that download model weights without declared source and license.

## Minimum run directory

```text
runs/{run_id}/
  inputs_manifest.json
  run_manifest.json
  logs/execution.log
  artifacts/
  report/
```

## Agent status vocabulary

| Status | Meaning |
|---|---|
| `unsupported` | Manifest or workflow cannot run on this machine. |
| `missing-inputs` | Required input files or parameters are missing. |
| `missing-environment` | Required tool/runtime is not available. |
| `missing-weights` | Required model weights are absent or unverified. |
| `ready-dry-run` | Dry run can be executed. |
| `ready-smoke-test` | Smoke test can be executed. |
| `ready-run` | User-provided inputs can be executed with approval. |
| `completed` | Run completed and run record was written. |
| `failed` | Run failed and logs were captured. |

## Non-goals

- The agent does not certify scientific truth.
- The agent does not infer clinical or drug-development meaning.
- The agent does not replace upstream documentation.
- The agent does not silently install arbitrary dependencies.
